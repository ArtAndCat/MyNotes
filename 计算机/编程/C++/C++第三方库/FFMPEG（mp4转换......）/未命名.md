在Android设备上直接编译FFmpeg并不常见，因为通常设备上的资源有限，不适合进行大型项目的编译。然而，如果您确实要在Android设备上本地编译FFmpeg，或者是指定如何为Android设备创建一个适配的FFmpeg二进制文件集合，则需要进行交叉编译。以下是针对Android设备的交叉编译FFmpeg的大致步骤：

1. **安装Android NDK**：
   - Android NDK提供了一套用于在非Android环境下编译产生能在Android设备上运行的ARM架构二进制文件的工具链。
   - 下载并安装适用于您开发环境的最新Android NDK。

2. **获取FFmpeg源代码**：
   - 从FFmpeg官方网站或其他可靠来源下载FFmpeg的源代码压缩包。

3. **配置交叉编译环境**：
   - 设置环境变量指向NDK的位置，比如：
     ```bash
     export NDK_ROOT=/path/to/your/android-ndk-rXX
     ```
   - 配置FFmpeg交叉编译选项，示例配置可能包括：
     ```bash
     ./configure \
       --prefix=$HOME/ffmpeg-android-install \
       --target-os=android \
       --cross-prefix=$NDK_ROOT/toolchains/llvm/prebuilt/linux-x86_64/bin/aarch64-linux-android- \
       --arch=aarch64 \
       --sysroot=$NDK_ROOT/sysroot \
       --extra-cflags="-I$NDK_ROOT/sysroot/usr/include" \
       --extra-ldflags="-L$NDK_ROOT/sysroot/usr/lib" \
       --enable-cross-compile \
       --enable-shared \
       --disable-static \
       --enable-small \
       --disable-doc \
       --disable-programs \
       --enable-protocol=file,http,https,tcp,udp,rtp,rtsp \
       --enable-demuxer=matroska,mp4,mov,avi \
       --enable-muxer=mp4,mov,matroska \
       --enable-decoder=vp8,vp9,h264,hevc,aac,mp3 \
       --enable-encoder=h264,aac \
       # 更多特定的编解码器、滤镜和其他选项...
     ```

4. **编译FFmpeg**：
   - 执行 `make` 命令来编译FFmpeg。
     ```bash
     make clean && make -j$(nproc)
     ```

5. **安装FFmpeg二进制文件**：
   - 执行 `make install` 将编译好的二进制文件和库安装到指定目录下。
     ```bash
     make install
     ```

6. **打包二进制文件集合**：
   - 将安装目录下的`bin`目录中的FFmpeg可执行文件（例如`ffmpeg`和`ffprobe`）以及其他必要的库文件收集起来，按照您的应用需求进行组织和打包。

7. **集成到ffmpeggui应用**：
   - 将打包好的二进制文件集合放到ffmpeggui应用内合适的目录结构中，并确保应用能够在运行时找到这些文件。

请注意，这里的示例配置针对的是64位ARM架构（aarch64），具体的配置选项应根据您的Android设备架构和需求进行调整。此外，对于Android应用而言，通常的做法是在PC端完成交叉编译，然后将编译好的FFmpeg库嵌入到APP中，而不是在Android设备上直接编译。
